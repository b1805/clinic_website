📩 Downloader Information

Quickly retrieve Supplementary Data, Papers and Datasets

What is the Downloader?

The Downloader is designed to automate the retrieval of supplementary files, research papers, and associated datasets from academic articles. It takes in a list of DOIs in CSV/Excel/Sheets format, systematically searches for supplementary materials across publishers and repositories, and organizes them into a structured, downloadable format.

How It Works

  • Input Stage
    • Upload a CSV or Excel or Sheets file containing DOIs (e.g., paper_list.csv).
    • The Downloader scans all columns, extracting DOIs using regex and filtering out everything else.
  • Retrieval Pipeline
    • Primary Method: Visits the DOI landing page, parses HTML, and searches for supplementary file links using keyword filters like “supplementary,” “dataset,” “S1,” etc.
    • Fallback APIs:
      • EuropePMC API – checks metadata and supplementary links.
      • Crossref API – retrieves metadata for associated datasets.
    • Domain-specific logic: Custom parsers handle publisher idiosyncrasies (e.g., Springer, Elsevier, Wiley). For Wiley in particular, automated API access is limited, so Selenium and API tokens are used, when possible, with manual fallback when necessary.
  • Download & Packaging
    • Files are streamed into temporary directories and checked to ensure they are not HTML pages mislabeled as datasets.
    • After collection, files are zipped together for clean download.
    • Logs are generated to track which DOIs succeeded, failed, or lacked supplementary content.
  • Papers Only Mode
    • A separate option allows you to download the paper PDF itself (currently via Wiley API integration).

Outputs

The Downloader returns results in ZIP archives and CSV logs. Depending on your run, the outputs may contain:

  • Supplementary datasets (CSV, XLSX, ZIP, etc.)
  • Paper PDFs (when using “Download Papers Only” mode)
  • Organized folder structure: each paper has its own folder, named by DOI and a cleaned paper title
  • A log file (CSV) recording the status of every DOI (success, missing supplements, failed retrieval, etc.)

Tips for Best Results

  • Be aware that some publishers (especially Wiley) restrict access to supplementary materials.
  • If retrieving broad sets of papers, expect variability: some papers have no supplements, others require manual downloads.

Example

  • Input: paper_list.csv containing 100 DOIs.
  • Process: Downloader scans each DOI → finds 65 papers with supplementary files, 20 with papers only, 15 with no downloadable data.
  • Output:
    • supplementary_data.zip → structured archive with per-paper folders (DOI_paperTitle/) containing CSVs, Excel, or ZIP files.
    • download_log.csv → status file summarizing each DOI’s outcome.
    • If “Download Papers Only” mode was used afterwards: PDF copies of each paper
